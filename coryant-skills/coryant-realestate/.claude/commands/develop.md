Development: $ARGUMENTS

Run DEVELOPMENT_STACK moment type.

MANDATORY SEQUENCE:

1. Red flag scan — title and environmental risks on the site first. Contamination, flood zone, planning constraints, access rights.

2. Planning intel — current use class/zoning, planning status (no consent / outline / full / prior approval), risk of achieving consent for proposed scheme, timeline to consent.

3. GDV calculation:
   - Search comparable residential sales or commercial lettings/yields for the proposed scheme
   - GDV = units × market capital value OR NIA × ERV / market yield
   - State GDV with confidence grade

4. Cost plan (reference benchmarks — state as [C] — must be verified with QS):
   - Build cost per sq ft by scheme type
   - Professional fees (12–15%)
   - Statutory fees and CIL/S106 (if UK)
   - Finance costs (land loan + build loan)
   - Contingency (10% minimum)
   - Marketing and sales
   - Developer profit (15–20% of GDV)

5. Residual Land Value = GDV − all costs − developer profit

6. Sensitivity table: GDV × build cost grid (3×3) showing residual land value

7. Viability verdict: does the scheme work at the proposed land price?
   - If residual > land price: viable — by how much?
   - If residual < land price: not viable — what would need to change (GDV up, costs down, density up, land price down)?
   - If residual is negative: scheme does not work regardless of land price — state what changes viability

Open with: GO / NO-GO / CONDITIONAL — with specific condition.
End with: "The single greatest risk to this scheme is: [specific]"

Produce output using mode-intel.md. State every cost benchmark as [C] and recommend QS pricing.
